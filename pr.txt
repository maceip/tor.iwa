## Title

Add Tor Hidden Service IWA ‚Äî WebMCP tools for anonymous AI agent networking

## Body

### Summary

This PR adds a **Tor Hidden Service running inside an Isolated Web App** with 6 WebMCP tools that give AI agents capabilities they can't get any other way: anonymous `.onion` browsing, peer-to-peer connections over Tor, and cryptographic access control for hidden services.

**Live demo**: An AI agent can `fetchOnion` to browse `.onion` sites through the user's Tor circuit, `holepunch` to open direct TCP connections to other hidden services for anonymous agent-to-agent communication, and `manageTrustedClients` to control who can access the hidden service with Ed25519 signature verification.

### Why this is interesting for WebMCP

This is the first WebMCP integration that gives AI agents access to **network-level capabilities** rather than just application features:

1. **`fetchOnion`** ‚Äî AI agents cannot normally reach `.onion` addresses. This tool routes requests through a real Tor circuit via Direct Sockets SOCKS5, returning HTTP responses with automatic TOFU certificate verification. The agent gets anonymous web access to the entire `.onion` ecosystem.

2. **`holepunch`** ‚Äî Opens real bidirectional TCP connections to other `.onion` hidden services. This is the foundation for **anonymous AI-to-AI communication** ‚Äî Agent A publishes a `.onion`, Agent B holepunches to it, they exchange messages without either knowing the other's IP address. Supports `connect` / `send` / `receive` / `close` actions on persistent sessions.

3. **`validateOnionCert`** ‚Äî Trust-on-first-use (TOFU) certificate management. Service fingerprints are auto-captured on every `fetchOnion` call and compared on subsequent fetches. Detects identity changes that may indicate service compromise. The AI agent manages its own trust decisions about anonymous services.

4. **`manageTrustedClients`** ‚Äî Real access control enforcement on the hidden service. When trusted clients are added, the HS HTTP handler rejects requests without a valid `X-Client-ID` header (returns 403). Supports Ed25519 public key registration and signature verification via `X-Client-Auth` headers.

5. **`getServiceStatus`** ‚Äî Aggregated dashboard of the entire system: HS uptime/request/byte stats, access control state, cert store health, active holepunch sessions, and recent fetch errors.

6. **`listHolepunchSessions`** ‚Äî Lists all peer connections with connection status, message/byte counts, and timestamps.

### Technical highlights

- **No build step** ‚Äî Pure ESM with Preact + htm, vendored dependencies
- **Real Tor** ‚Äî `tor.js` + `tor.wasm` compiled via Emscripten, connects to the actual Tor network
- **Real TCP** ‚Äî `TCPSocket` for SOCKS5 outbound, `TCPServerSocket` for HS inbound (Direct Sockets API)
- **OPFS keypair persistence** ‚Äî `.onion` address survives page reloads via Origin Private File System
- **Web Locks** ‚Äî Prevents multiple tabs from fighting over the HS listener port
- **Ed25519 via Web Crypto** ‚Äî `.onion` v3 address derivation (SHA3-256 checksum) and client auth signatures
- **OHTTP framing demo** ‚Äî Correct RFC 9458 Binary HTTP encoding with AES-GCM, ready for a real relay
- **Visibility-aware animations** ‚Äî Canvas loops pause when tab is hidden to save battery
- **O(n) buffer handling** ‚Äî Chunk-then-concat pattern for SOCKS5/HTTP response reading

### Browser API surface

| API | Usage |
|-----|-------|
| Direct Sockets | `TCPSocket` (SOCKS5), `TCPServerSocket` (HS listener) |
| WebMCP | `navigator.modelContext.registerTool()` x 6 |
| Web Crypto | Ed25519 keygen/sign/verify, AES-GCM, ECDH, SHA-256 |
| OPFS | `navigator.storage.getDirectory()` for keypair persistence |
| Web Locks | `navigator.locks.request()` for HS singleton |
| File System Access | `showDirectoryPicker()` for Tor data dir |
| Service Worker | Offline asset caching |
| Trusted Types | CSP compliance (IWA requirement) |

### What an AI agent can do with these tools

```
Agent: "I want to check if the DuckDuckGo onion service is up"
‚Üí fetchOnion({ url: "http://duckduckgogg42xjoc72x3sjasowoarfbgcmvfimaftt6twagswzczad.onion/" })
‚Üí { success: true, status: 200, certVerification: { known: false, note: "First contact ‚Äî fingerprint auto-stored via TOFU" } }

Agent: "Connect to another agent's hidden service"
‚Üí holepunch({ action: "connect", targetOnion: "abc123...xyz.onion", port: 80 })
‚Üí { success: true, sessionId: "...", status: "connected" }

Agent: "Send a message"
‚Üí holepunch({ action: "send", sessionId: "...", message: '{"hello": "world"}' })
‚Üí { success: true, bytesSent: 22 }

Agent: "Lock down my hidden service to only trusted clients"
‚Üí manageTrustedClients({ action: "add", clientId: "agent-b", pubkey: "base64..." })
‚Üí { success: true, note: "Access control is now ACTIVE" }
```
